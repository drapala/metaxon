# Obsidian as Agent Wiki — Kepano's Recommended Setup

More and more people are using Obsidian as a local wiki to read things your agents are researching and writing. It works best with a separate Obsidian vault that you can fill with content, e.g. via Obsidian Web Clipper.

## The Four Pieces

1. **Obsidian app** — the local-first markdown editor
2. **Obsidian Web Clipper** extension — capture content in .md format
3. **Obsidian CLI** — so agents can interact with the full feature set of the app (e.g. backlinks, bases, etc)
4. **Obsidian Skills** — so agents know how to create .md, .base and .canvas

## Vault Separation

> I like @karpathy's Obsidian setup as a way to mitigate contamination risks. Keep your personal vault clean and create a messy vault for your agents.

> I prefer my personal Obsidian vault to be high signal:noise, and for all the content to have known origins.

## Human-First Philosophy

> Obsidian is human-first. You can explore the Obsidian website and app without ever encountering the word "AI" or a sparkly button.

> If you want AI in your vault you can do so easily using plugins or just the interoperability that plain text provides.
