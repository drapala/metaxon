# Oblique Strategies: Over One Hundred Worthwhile Dilemmas

## Core Concept

Card-based method for promoting creativity through lateral thinking. Each card contains a gnomic suggestion meant to break deadlocks. Created 1975 by Brian Eno and Peter Schmidt.

## Mechanism

Intentional introduction of randomness into creative process. Disrupts linear thinking. Forces exploration of unconventional angles. Creates unexpected connections.

Key principle: "Honor thy error as a hidden intention" — reframe problems, don't solve them head-on.

## Selected Strategies Relevant to KB Question Design

| Strategy | KB Application |
|----------|---------------|
| "What would your closest friend do?" | Ask from a different persona's perspective |
| "Use an old idea" | Apply a foundational concept (Zettelkasten, Shannon) to modern problem |
| "What is the reality of the situation?" | Strip away interpretation, look at raw/ only |
| "Reverse" | Invert the question: "when does this NOT work?" |
| "Remove specifics and convert to ambiguities" | Abstract a specific finding to general principle |
| "What mistakes did you make last time?" | Review previous /ask failures before asking again |
| "Emphasize differences" | Instead of finding convergence, find divergence between sources |
| "Do nothing for as long as possible" | Don't ask yet — let the wiki sit and see what questions emerge naturally |
| "Look closely at the most embarrassing details" | Focus on the weakest claim, not the strongest |

## Why It Matters for KBs

Default /ask behavior: user asks direct question → LLM answers directly. This is linear thinking.

Oblique approach: before answering, REFRAME the question. "What if the opposite were true?" "What's the most boring version of this question?" "What would someone who disagrees with the entire premise ask?"

The value is in the question, not the answer. A KB with 42 sources can answer any direct question. The competitive advantage is asking questions nobody else would ask.
