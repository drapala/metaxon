# Deep Research: A Survey of Autonomous Research Agents

## Abstract

Survey examining how LLMs power autonomous research agents that overcome knowledge limitations. Describes a systematic pipeline with four stages: planning, question development, web exploration, and report generation. Analyzes technical challenges at each stage, reviews optimization techniques and benchmarks designed for deep research systems, and outlines future directions.

## The Four-Stage Deep Research Pipeline

### 1. Planning

Transforming a research question and prior knowledge into a structured research plan consisting of intermediate subgoals or actions.

**Key Approaches:**

| Category | Methods |
|----------|---------|
| Planning with Structured World Knowledge | WebDreamer, Simulate Before Act, WebPilot, WKM, Thought of Search |
| Planning as Learnable Process | AgentSquare, Agent-E, self-training approaches, MindSearch, SimpleDeepSearcher |

**Technical Focus:**
- World model simulation to mentally roll out candidate trajectories
- Modular pipeline design with specialized sub-agents
- Efficiency-aware strategies balancing reasoning depth against resource usage
- Meta-learning approaches for adaptive planning across diverse environments

**Challenges:**
- Plans generated by current LLMs lack robustness to ambiguous questions
- Internal consistency not guaranteed; hallucinated steps propagate errors downstream
- Evaluation remains coarse, relying on end-task accuracy rather than plan quality
- Limited ability to leverage shared structures across tasks

### 2. Question Developing

Converting each subgoal within a structured plan into search queries, guided by the overall plan, current subgoal, and accumulated evidence.

**Key Approaches:**

| Optimization Type | Representative Methods |
|-------------------|----------------------|
| Reward-Optimized | DeepResearcher, EvolveSearch, R1-Searcher, Search-R1, ZeroSearch, DeepRetrieval |
| Multi-Dimensional Rewards | InForage, OTC-PO, IKEA, AutoRefine, R-Search, MMSearch-R1, VRAG-RL |
| Supervision-Driven | ManuSearch, Search-o1, SearchAgent-X, ReasonRAG |

**Reward-Optimized:** RL to optimize query generation through interaction with search environment. Format rewards enforce syntactic validity; answer rewards (F1-based) improve retrieval relevance. Multi-dimensional rewards incorporate information gain, efficiency penalties, and knowledge boundary awareness.

**Supervision-Driven:** Structured multi-agent systems with separate agents for planning, searching, and extraction. Imitation learning via MCTS and DPO. More controllable and interpretable but limited by demonstration quality.

**Challenges:**
- Generated queries rely too heavily on clarity of subgoals
- Lack of contextual coherence; repetitive or redundant questions
- Poor performance on open-ended problems; overly narrow queries limit exploratory capabilities

### 3. Web Exploration

Retrieving the most relevant information from online sources through autonomous agents or search engine APIs.

#### Web Retrieval Agents

**Browser-Based:** WebGPT pioneered text-based browsers. Selenium enables real browser interaction via DOM. Limitation: fail when critical information appears in visual layouts.

**Multimodal:** MM-ReAct, WebVoyager, WebArena combine rendered screenshots with textual metadata. WebVoyager achieved 59% task success on real-world benchmarks with GPT-4V.

#### API-Based Retrieval Systems

| System | Infrastructure |
|--------|---------------|
| OpenAI DeepResearch | Microsoft Bing web-search |
| Grok DeepSearch | xAI crawler + X posts access |
| Gemini DeepResearch | Google proprietary search |
| Perplexity DeepResearch | Bing-style index + Sonar API |
| Domain-Specific | CNKI, Reportify, BraveSearch |

Web agents excel at dynamically navigating complex interfaces. API systems ensure reliability but lack adaptability for specialized queries. Future: hybrid architectures combining both with advanced content verification.

### 4. Report Generation

Synthesizing comprehensive reports integrating web information into coherent, logically organized, fact-grounded outputs.

#### Structure Control

| Approach | Methods |
|----------|---------|
| Planning-Based | Agent Laboratory, AI Scientist v2, LongEval, LongWriter, LongDPO |
| Constraint-Guided | WebThinker, Suri, human-like revision frameworks |
| Structure-Aware Evaluation | Long2RAG, ExPerT, attribute-guided alignment |

#### Factual Integrity

| Category | Methods |
|----------|---------|
| Faithful Modeling | RAGSynth, BRIDGE, context-aware decoding |
| Conflict Resolution | FaithfulRAG, DRAGged, entropy-based decoding |
| Factuality Evaluation | Face4RAG, SFR-RAG, FaithJudge, RAG-QA Arena, MT-RAIG |

**Current Limitations:** Most methods target isolated subskills while lacking joint optimization. Fixed outline strategies. Few approaches model consistency across conflicting sources.

## Workflow Architectures

**Single-Agent:** Consolidated LLM responsible for entire process. Examples: DeepResearcher, WebThinker, Search-R1. Advantage: integrated end-to-end learning.

**Multi-Agent:** Specialized agents for specific stages. Examples: AgentRxiv, AI Scientist, OpenResearcher. Advantages: independent optimization, parallelism, flexibility.

### Parameter Optimization

| Method | Approach |
|--------|----------|
| Contrastive Learning | Avatar contrasts effective/ineffective behaviors in tool usage |
| Reinforcement Learning | Search-O1, Learning-to-Search optimize via reward signals |
| Curriculum Training | AI Scientist-v2, SimpleDeepSearcher use staged learning pipelines |

## Benchmarks

### Search-Oriented
- Mind2Web 2, BrowseComp, WebArena, GAIA
- Focus: information-seeking with Web Exploration and Question Developing
- Rarely involve final report generation

### Research-Oriented

| Benchmark | Coverage | Key Metrics |
|-----------|----------|-------------|
| DeepResearch Bench | P, QD, WE, RG | Pairwise Agreement Rate, Pearson Correlation |
| DeepResearchGym | P, QD, WE, RG | KPR, KPC, Precision, Recall, Clarity, Insight |
| MedBrowseComp | P, QD, WE, RG | Accuracy (biomedical domain) |
| Humanity's Last Exam | P, QD, WE | Accuracy, Calibration Error |

## Technical Challenges

1. **Multi-Tool Integration:** Most systems rely solely on search engines; lack support for APIs, databases, code repos, structured data
2. **Factuality:** Agents introduce inaccuracies, outdated claims, unsupported assertions when aggregating inconsistent sources
3. **Multimodal Reasoning:** Pipelines remain textual; unsuitable for science, medicine, engineering requiring visual analysis
4. **Workflow Design:** Lack mechanisms for learning generalizable workflows; high computational costs and sparse reward signals
5. **Personalization:** No persistent user modeling or dynamic adaptation

## Future Directions

- Richer tool orchestration (APIs, databases, repositories, structured documents)
- Stronger factual grounding (source attribution, factuality-aware rewards, post-hoc verification)
- Multimodal extension (images, PDFs, tables, structured data)
- Efficient agent training (parameter-efficient tuning, curriculum pretraining, human/automated feedback RL)
- Cross-task generalization (transferable planning knowledge, adaptive workflows)

## Conclusion

Deep research represents a shift from passive retrieval toward active, goal-driven knowledge synthesis. Four interconnected stages—planning, question developing, web exploration, and report generation—integrate modular competencies to produce comprehensive, faithful research outputs.
